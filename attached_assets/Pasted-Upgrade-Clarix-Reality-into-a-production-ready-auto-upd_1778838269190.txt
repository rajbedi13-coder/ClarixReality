Upgrade Clarix Reality into a production-ready, auto-updating premium intellectual intelligence platform.

Important:
Do not break the current working website. Keep the existing design and features that already work, but improve them into a more premium, scalable, production-ready version.

Core vision:
Clarix Reality should feel like a cinematic, editorial, premium digital intelligence platform — not a basic AI news dashboard, not a generic SaaS app, and not a cheap AI-generated news site.

The platform should be inspired by:
- Geobarta for atmosphere and intellectual feed style
- Bloomberg and Reuters for credibility and source discipline
- The Economist for editorial intelligence
- Aeon Essays for philosophical and psychological depth
- modern AI discovery feeds for personalisation

Brand positioning:
Clarix Reality is a “signal over noise” platform for geopolitics, markets, psychology, philosophy, civilisation, technology, and world intelligence.

Main tagline:
“Signal over noise for geopolitics, markets, psychology and civilisation.”

Main objective:
Build the platform so it can support real, auto-updating content from RSS/news APIs, AI summarisation, database storage, admin review, and dynamic publishing.

Do not simply add more fake sample cards. Build the architecture properly so real content can later flow through this system:

Sources/RSS/API
→ AI summarisation
→ database
→ admin review
→ published feed
→ user interaction/personalisation

Legal and credibility rules:
- Do not copy full articles.
- Only use title, source, URL, metadata, short AI summary, key facts, and “why it matters”.
- Always show original source name and source link.
- Every quote, historical claim, philosophical reference, and factual claim must include a source/reference field.
- Avoid fake “live” labels unless content is truly live.
- Avoid clickbait, gossip, meme content, rage bait, low-quality viral content, and unsupported claims.
- Clearly show that summaries are AI-assisted and source-transparent.

Backend/content pipeline:
Create a scalable content ingestion system with the following structure:

1. RSS/news API ingestion
- Pull articles from trusted RSS feeds or news APIs.
- Support multiple sources.
- Allow sources to be added/removed later.
- Check for new content on a schedule.

2. Duplicate prevention
Avoid duplicate articles using:
- source URL
- title similarity
- publish date
- source name

3. AI summary structure
For each article, generate/store:
- headline
- short summary
- why it matters
- key facts
- category
- tags
- impact level
- reading time
- source name
- original source link
- publish date
- image thumbnail if available
- content type
- review status

4. Database-ready architecture
Store content dynamically instead of hardcoding article cards.
If a real database is not connected yet, create a clean mock database/data layer that can later be replaced with Supabase/PostgreSQL.

5. Admin review system
Create an admin dashboard with:
- pending articles
- approved articles
- rejected articles
- edit summary
- edit category
- edit tags
- delete article
- approve/publish button
- manual approval mode
- trusted-source auto-publish mode

Default mode should be manual approval, not full auto-publish.

6. Scheduled updates
Add scheduled-job-ready structure for checking new articles every few hours.
If real cron/scheduler is not available in the current environment, create the functions and explain where the scheduler should connect later.
